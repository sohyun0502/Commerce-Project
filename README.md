# 🛒 상품 관리 시스템 (Commerce Management System)

Java 콘솔 환경에서 동작하는 **상품 관리 및 구매 시스템**입니다.  
관리자와 고객 역할을 분리하여,
- **관리자**는 상품과 카테고리를 관리하고
- **고객**은 상품을 조회하고 장바구니를 통해 주문을 진행하며  
  주문 금액 누적에 따라 **고객 등급이 자동으로 변경**되도록 설계했습니다.

---

## 📌 프로젝트 개요

- **프로젝트 유형**: 콘솔 기반 Java 애플리케이션
- **주요 목적**
  - 관리자 / 고객 역할 분리 설계
  - 상품 및 카테고리 관리 기능 구현
  - 장바구니 기반 주문 처리
  - 누적 주문 금액 기반 고객 등급 자동 변경
  - 객체지향 프로그래밍(OOP) 설계 연습

---

## 🛠 기술 스택

- **Language**: Java
- **Input 방식**: Scanner (Console Input)
- **설계 방식**: 객체지향 설계 (OOP)
- **Java 기능**: Enum, Stream, Lambda, Predicate

---

## 📂 프로젝트 구조

```
commerce/
 ├── Administrator.java     # 관리자 기능 (상품 CRUD)
 ├── Customer.java          # 고객 정보 및 누적 금액 관리
 ├── CustomerLevel.java     # 고객 등급(Enum, 할인율 및 기준)
 ├── CommerceSystem.java    # 전체 시스템 흐름 제어
 ├── Category.java          # 카테고리 및 상품 목록 관리
 ├── Product.java           # 상품 정보
 ├── Cart.java              # 장바구니 및 주문 관리
 └── Main.java              # 프로그램 실행 진입점
```

## 👤 사용자 역할

### 🔐 관리자 (Administrator)

- 비밀번호 인증 기반 접근
- 상품 관리 전용 기능 제공

#### 관리자 인증
- 비밀번호: `admin123`
- **3회 실패 시 메인 메뉴로 이동**

#### 관리자 기능
- 상품 추가
- 상품 수정
- 상품 삭제
- 전체 상품 조회

---

### 🧑 고객 (Customer)

- 이메일 기반 고객 조회
- 상품 조회 및 장바구니 사용
- 주문 확정 시 금액 누적
- 누적 금액에 따라 고객 등급 자동 변경

#### 고객 등급 정보

| 누적 주문 금액 | 등급 | 할인율 |
|---|---|---|
| 500,000원 미만 | BRONZE | 0% |
| 500,000원 이상 ~ 1,000,000원 미만 | SILVER | 5% |
| 1,000,000원 이상 ~ 2,000,000원 미만 | GOLD | 10% |
| 2,000,000원 이상 | PLATINUM | 15% |

> 고객 등급 변경 로직은  
> `CustomerLevel` Enum 내부에 캡슐화하여 관리합니다.

## 📦 상품 관리 기능 (관리자 전용)

### 1️⃣ 상품 추가
- 카테고리 선택
- 상품 정보 입력
    - 상품명
    - 가격
    - 설명
    - 재고 수량
- 입력 정보 확인 후 등록

### 2️⃣ 상품 수정
- 상품명으로 검색
- 수정 가능 항목
    - 가격
    - 설명
    - 재고 수량

### 3️⃣ 상품 삭제
- 상품명 기준 삭제
- 삭제 시 **장바구니에 담긴 상품도 함께 제거**

### 4️⃣ 전체 상품 조회
- 카테고리별 상품 목록 출력
- 상품이 없는 경우 안내 메시지 출력

---

## 🛒 고객 장바구니 및 주문 기능

### 1️⃣ 상품 조회
- 카테고리별 상품 목록 조회
- 가격대별 필터링 기능
  - 100만원 이하 상품
  - 100만원 초과 상품
- Stream + Predicate 기반 필터링 구현

### 2️⃣ 장바구니 담기
- 조회한 상품을 선택하여 장바구니에 추가
- 동일 상품 중복 선택 가능

### 3️⃣ 장바구니 확인
- 현재 장바구니에 담긴 상품 목록 출력
- 상품별 가격 및 수량 확인

### 4️⃣ 주문 확정
- 고객 이메일 입력을 통한 고객 조회
- 고객 등급에 따른 할인율 안내
- 주문 확정 여부 선택
- 주문 확정 시:
  - 할인 금액 계산
  - 최종 결제 금액 산출
  - 고객 누적 주문 금액에 반영
  - 누적 금액 기준 고객 등급 자동 변경

---

## 💡 주요 구현 포인트

### ✔ 역할 분리
- `Administrator`: 관리자 인증 및 상품 관리
- `Customer`: 고객 정보, 누적 금액, 등급 변경
- `CustomerLevel`: 등급 기준 및 할인 정책
- `Category`: 카테고리별 상품 관리
- `Product`: 상품 데이터 및 출력 책임
- `Cart`: 장바구니 및 주문 상태 관리
- `CommerceSystem` : 전체 흐름 제어

### ✔ Enum 활용
- 고객 등급을 Enum으로 관리하여
- 할인율 변경 시 유지보수 용이
- 가독성과 안정성 향상

### ✔ Stream & Lambda 활용
- 상품 필터링 로직에 `Stream`, `Predicate` 적용
- 반복문 최소화 및 가독성 향상

### ✔ 데이터 일관성
- 상품 삭제 시 장바구니에서도 자동 제거
- 장바구니 상태에 따라 메뉴 노출 제어
