# 🛒 상품 관리 시스템 (Commerce Management System)

Java 콘솔 환경에서 동작하는 **상품 관리 및 구매 시스템**입니다.  
관리자와 고객 역할을 분리하여,
- 관리자는 상품을 관리하고
- 고객은 등급을 선택하여 할인 혜택을 받을 수 있도록 설계되었습니다.

객체지향 설계를 기반으로 **상품, 카테고리, 장바구니, 고객, 관리자**의 책임을 분리하여 구현했습니다.

---

## 📌 프로젝트 개요

- **프로젝트 유형**: 콘솔 기반 Java 애플리케이션
- **주요 목적**
    - 상품 및 카테고리 관리 기능 구현
    - 관리자 권한 기반 상품 CRUD
    - 고객 상품 조회 및 장바구니 기능 구현
    - 고객 등급(Enum) 기반 할인 정책 적용
    - 객체지향 프로그래밍(OOP) 설계 연습

---

## 🛠 기술 스택

- **Language**: Java
- **Input 방식**: Scanner (Console Input)
- **설계 방식**: 객체지향 설계 (OOP)
- **Java 기능**: Enum, Stream, Lambda, Predicate

---

## 📂 프로젝트 구조

```
commerce/
 ├── Administrator.java     # 관리자 기능 (상품 CRUD)
 ├── Customer.java          # 고객 정보 및 등급 선택
 ├── CustomerLevel.java     # 고객 등급(Enum, 할인율 관리)
 ├── CommerceSystem.java    # 전체 시스템 흐름 제어
 ├── Category.java          # 카테고리 및 상품 목록 관리
 ├── Product.java           # 상품 정보
 ├── Cart.java              # 장바구니 및 주문 관리
 └── Main.java              # 프로그램 실행 진입점
```

## 👤 사용자 역할

### 🔐 관리자 (Administrator)

- 비밀번호 인증 기반 접근
- 상품 관리 전용 기능 제공

#### 관리자 인증
- 비밀번호: `admin123`
- **3회 실패 시 메인 메뉴로 이동**

---

### 🧑 고객 (Customer)

- 고객 등급 선택
- 등급에 따라 할인율 적용

#### 고객 등급 선택
```
1. BRONZE : 0% 할인
2. SILVER : 5% 할인
3. GOLD   : 10% 할인
4. VIP    : 15% 할인
```

> 고객 등급은 `CustomerLevel` Enum으로 관리되며  
> 각 등급은 고정된 할인율을 가집니다.

---

## 📦 상품 관리 기능 (관리자 전용)

### 1️⃣ 상품 추가
- 카테고리 선택
- 상품 정보 입력
    - 상품명
    - 가격
    - 설명
    - 재고 수량
- 입력 정보 확인 후 등록

### 2️⃣ 상품 수정
- 상품명으로 검색
- 수정 가능 항목
    - 가격
    - 설명
    - 재고 수량

### 3️⃣ 상품 삭제
- 상품명 기준 삭제
- 삭제 시 **장바구니에 담긴 상품도 함께 제거**

### 4️⃣ 전체 상품 조회
- 카테고리별 상품 목록 출력
- 상품이 없는 경우 안내 메시지 출력

---

## 🧾 출력 예시

```
[ 전자제품 ]
노트북 | 1,200,000원 | 고성능 노트북 | 재고: 5개

[ 식품 ]
등록된 상품이 없습니다.
```

---

## 🛒 고객 장바구니 및 주문 기능

### 1️⃣ 상품 조회
- 카테고리별 상품 목록 조회
- 가격대별 필터링 기능
  - 100만원 이하 상품
  - 100만원 초과 상품
- Stream + Predicate 기반 필터링 구현

### 2️⃣ 장바구니 담기
- 조회한 상품을 선택하여 장바구니에 추가
- 동일 상품 중복 선택 가능

### 3️⃣ 장바구니 확인
- 현재 장바구니에 담긴 상품 목록 출력
- 상품별 가격 및 수량 확인

### 4️⃣ 주문 취소
- 진행 중인 주문(장바구니) 전체 취소
- 장바구니 초기화

> 장바구니는 `Cart` 객체에서 관리되며  
> 상품 삭제 시 장바구니와의 **데이터 일관성**을 유지합니다.

---

## 🧾 출력 예시

```
[ 전자제품 ]
노트북 | 1,200,000원 | 고성능 노트북 | 재고: 5개

[ 장바구니 ]
노트북 x 1 | 1,200,000원

총 주문 금액: 1,200,000원
```

## 💡 주요 구현 포인트

### ✔ 역할 분리
- `Administrator`: 관리자 인증 및 상품 관리
- `Customer`: 고객 등급 선택
- `CustomerLevel`: Enum 기반 할인 정책
- `Category`: 카테고리별 상품 관리
- `Product`: 상품 데이터 및 출력 책임
- `Cart`: 장바구니 및 주문 상태 관리

### ✔ Enum 활용
- 고객 등급을 Enum으로 관리하여
    - 할인율 변경 시 유지보수 용이
    - 가독성과 안정성 향상

### ✔ Stream & Lambda 활용
- 상품 필터링 로직에 `Stream`, `Predicate` 적용
- 반복문 최소화 및 가독성 향상

### ✔ 데이터 일관성
- 상품 삭제 시 장바구니에서도 자동 제거
- 장바구니 상태에 따라 메뉴 노출 제어
